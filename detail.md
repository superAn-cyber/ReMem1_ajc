# ReMemR1 论文流程详解

> 基于代码（`recurrent/impls/memory_revisit.py`、`recurrent/impls/tf_idf_retriever.py`）的逐步解析。

---

## 一、论文解决的根本问题

假设有一篇 **50,000 token 的长文档**，LLM 的上下文窗口只有 8,000 token，一次放不下整篇文章。

最直觉的做法是：把文章切成块，**一块一块读，边读边记笔记（memory）**，读完再回答问题。这就是 **MemAgent（baseline）** 的做法。

**MemAgent 的致命问题**：每次更新记忆时，新信息会覆盖旧信息，导致早期读到的关键内容被遗忘。

**ReMemR1 的核心创新**：允许模型在读到后面的内容时，**主动回头翻之前的笔记**（Callback/Recall 机制）。

---

## 二、三个核心数据结构

```
memory          ← 当前最新的笔记（字符串，每步覆盖更新）
history_memory  ← 历史所有笔记的集合（Python Set，只增不减）
recalled_memory ← 本步从历史里检索到的最相关笔记（TF-IDF 检索结果）
```

对应代码位置（`memory_revisit.py`）：

| 概念 | 代码位置 |
|---|---|
| memory 初始化 | `MemoryAgent.start()` 里的 `self.memory` |
| history_memory 存入 | `update_memory()` 方法 |
| recall 查询解析 | `_parse_recall_query()` |
| TF-IDF 检索历史 | `tf_idf_retriever.py` 的 `top1_retrieve()` |
| memory 内容解析 | `_parse_update_memory()` |
| 最终回答 | `is_final = True` 后切换为 `TEMPLATE_FINAL_BOXED` |

## 三、完整流程举例

### 任务设定

**问题**：文章中提到的这位考生，他的目标院校复试线是多少？他的初试成绩有没有希望过线？

**文档**（实际上是 50,000 token 的长篇备考资料，这里极度简化为 4 块）：

```
[第1块] 李明是一名计划报考研究生的大四学生，本科就读于某211大学软件工程专业。
        根据他的模拟考试成绩估算，预计初试总分约为 362 分。
        他对自然语言处理方向十分感兴趣。

[第2块] 近年来，人工智能领域快速发展，各高校相关专业报考人数持续增加。
        以计算机科学与技术专业为例，2024 年全国报考人数较 2020 年增长了约 40%。

[第3块] 华中科技大学计算机学院 2024 年学硕复试基本线为：
        总分 355 分，政治单科 48 分，英语单科 48 分，数学 100 分，专业课 100 分。
        李明此前透露，他的第一志愿正是华中科技大学计算机学院。

[第4块] 华中科技大学计算机学院 2024 年实际录取统考生 24 人，
        录取的统考生中最低分为 373 分，平均分为 391 分。
```

这是一道**多跳推理题**：要回答"李明有没有希望过线"，需要把"李明预计 362 分"（第1块）和"华科复试线 355 分"（第3块）跨块连接起来对比。

---

### Step 0：初始状态

```python
memory          = "暂无记忆"
history_memory  = set()   # 空集合，存放历史上每一步的记忆快照
recalled_memory = "未召回任何记忆"
```

---

### Step 1：读第 1 块

**输入给 LLM 的 Prompt**（对应代码中的 `TEMPLATE`）：

```
你被给予一个问题、一段文章片段和之前的记忆。
请更新记忆，如有需要可以输出 <recall>查询内容</recall>。

<problem>
文章中提到的这位考生，他的目标院校复试线是多少？他的初试成绩有没有希望过线？
</problem>

<recalled_memory>未召回任何记忆</recalled_memory>

<memory>暂无记忆</memory>

<section>
李明是一名计划报考研究生的大四学生，本科就读于某211大学软件工程专业。
根据他的模拟考试成绩估算，预计初试总分约为 362 分。
他对自然语言处理方向十分感兴趣。
</section>
```

**LLM 输出**：

```
<thinking>
  这段介绍了考生李明，预计初试 362 分，与问题直接相关，记下来。
  目前还不知道他的目标院校，继续往后读。
</thinking>
<update>考生李明，预计初试总分 362 分，本科 211 软件工程，方向是 NLP，目标院校待确认。</update>
```

**代码执行**（`update()` 方法）：

```python
# 解析 <recall> 标签 → 无
recall_query = None
recalled_memory = "未召回任何记忆"

# 解析 <update> 标签内容，更新当前记忆
memory = "考生李明，预计初试总分 362 分，本科 211 软件工程，方向是 NLP，目标院校待确认。"

# 存入历史集合（Set 只增不减）
history_memory.add(memory)
# history_memory = {"考生李明，预计初试总分 362 分..."}
```

---

### Step 2：读第 2 块

**输入给 LLM**：

```
<memory>考生李明，预计初试总分 362 分，本科 211 软件工程，目标院校待确认。</memory>
<recalled_memory>未召回任何记忆</recalled_memory>
<section>
近年来，人工智能领域快速发展，各高校相关专业报考人数持续增加。
以计算机科学与技术专业为例，2024 年全国报考人数较 2020 年增长了约 40%。
</section>
```

**LLM 输出**：

```
<thinking>这段是背景信息，与具体考生关系不大，简单补充即可。</thinking>
<update>考生李明，预计初试总分 362 分，本科 211 软件工程，目标院校待确认。
背景：2024 年计算机专业报考人数较 2020 年增长约 40%，竞争激烈。</update>
```

**执行后状态**：

```python
memory = "考生李明，预计 362 分，本科 211。2024 年计算机报考人数增长 40%。"
history_memory = {
    "考生李明，预计初试总分 362 分...",   # Step 1 的快照
    "考生李明，预计 362 分...报考增长 40%"  # Step 2 的快照
}
```

---

### Step 3：读第 3 块 ← **Recall 触发的关键步骤**

**输入给 LLM**：

```
<memory>考生李明，预计 362 分，本科 211。2024 年计算机报考人数增长 40%。</memory>
<recalled_memory>未召回任何记忆</recalled_memory>
<section>
华中科技大学计算机学院 2024 年学硕复试基本线为：
总分 355 分，政治单科 48 分，英语单科 48 分，数学 100 分，专业课 100 分。
李明此前透露，他的第一志愿正是华中科技大学计算机学院。
</section>
```

**LLM 输出**（这一步触发了 Recall！）：

```
<thinking>
  这段说了两件事：
  1. 华科计算机复试线总分 355 分
  2. 李明的目标院校就是华科计算机

  问题要我判断李明"有没有希望过线"，需要把他的初试成绩和复试线做对比。
  我当前的 memory 里有提到"362 分"，但我想从历史记忆里明确确认李明的具体分数，
  避免记忆更新过程中数字被模糊掉。
</thinking>
<recall>李明的预计初试成绩是多少分</recall>
<update>李明目标院校：华科计算机学院，复试线总分 355 分（政治 48，英语 48，数学 100，专业课 100）。
李明预计 362 分，高于复试线 355 分，理论上可以过线参加复试。</update>
```

**代码执行**（关键！）：

```python
# 1. 解析 <recall> 标签
recall_query = "李明的预计初试成绩是多少分"

# 2. 用 TF-IDF 在 history_memory 集合里搜索最相关的历史记忆
#    直接调用 tf_idf_retriever.py 的 top1_retrieve()
recalled = retriever.top1_retrieve(query=recall_query, corpus=history_memory)
# 返回: "考生李明，预计初试总分 362 分，本科 211 软件工程，目标院校待确认。"

# 3. 把召回结果存入 recalled_memory，下一步 prompt 里会用到
recalled_memory = recalled

# 4. 解析 <update>，覆盖更新当前记忆
memory = "李明目标华科计算机，复试线 355 分。李明预计 362 分 > 355 分，理论上可以过线。"

# 5. 把新记忆存入历史集合
history_memory.add(memory)
```

**此时状态**：

```python
memory          = "李明目标华科计算机，复试线 355 分。李明预计 362 分 > 355 分，理论上可以过线。"
history_memory  = {Step1快照, Step2快照, Step3快照}   # 集合里保存了3条历史
recalled_memory = "考生李明，预计初试总分 362 分，本科 211 软件工程，目标院校待确认。"
```

---

### Step 4：读第 4 块

**输入给 LLM**（注意 `recalled_memory` 已经有内容了）：

```
<memory>李明目标华科计算机，复试线 355 分。李明预计 362 分 > 355 分，理论上可以过线。</memory>
<recalled_memory>考生李明，预计初试总分 362 分，本科 211 软件工程，目标院校待确认。</recalled_memory>
<section>
华中科技大学计算机学院 2024 年实际录取统考生 24 人，
录取的统考生中最低分为 373 分，平均分为 391 分。
</section>
```

**LLM 输出**：

```
<thinking>
  这段说明了实际录取情况：最低分 373 分，而李明只有 362 分。
  虽然李明能过复试线（355），但实际上 362 低于最低录取分 373，竞争压力很大。
  这是重要信息，需要更新记忆并补充这一关键差距。
</thinking>
<update>李明预计 362 分，华科计算机复试线 355 分（可以过线），但 2024 年实际最低录取分为 373 分，
李明分数低于最低录取线 11 分，录取风险较高。</update>
```

---

### Final Step：所有块读完，回答问题

代码检测到 `ctx_length <= step * chunk_size`，将 `is_final = True`，切换为 `TEMPLATE_FINAL_BOXED`：

```
你被给予一个问题和之前积累的记忆，请基于记忆回答，答案放在 \boxed{} 中。

<problem>
文章中提到的这位考生，他的目标院校复试线是多少？他的初试成绩有没有希望过线？
</problem>

<recalled_memory>考生李明，预计初试总分 362 分，本科 211 软件工程。</recalled_memory>

<memory>李明预计 362 分，华科计算机复试线 355 分（可以过线），但 2024 年实际最低录取分为 373 分，
李明分数低于最低录取线 11 分，录取风险较高。</memory>
```

**LLM 最终输出**：

```
\boxed{华科计算机复试线为 355 分，李明预计 362 分可以过线，但实际最低录取分为 373 分，录取存在一定风险。}
```

## 四、MemAgent vs ReMemR1 对比

| | MemAgent（baseline）| ReMemR1 |
|---|---|---|
| 读文档方式 | 线性，一块一块往前走 | 线性读，但可以**主动回头** |
| history_memory | 无，每步只保留最新记忆 | 保存所有历史快照（Set） |
| Recall 机制 | 无 | 有，用 TF-IDF 检索历史 |
| 早期信息遗忘 | 严重 | 通过 recall 缓解 |
| 模型训练 | RL 训练更新记忆行为 | RL 训练同时学习 recall 时机 |

---

## 五、RL 训练的作用

上面的流程在**推理阶段**是这样工作的，但论文的核心是用 **RL（GRPO）训练模型**，让模型自己学会：

1. **何时触发 `<recall>`**：不是每步都 recall，只在真正需要历史信息时触发
2. **recall 什么内容**：学会写出高质量的检索 query
3. **如何更新 memory**：提炼关键信息，不冗余、不遗漏

**奖励信号设计**（Multi-Level Reward，来自 `reward_design.png`）：
- 最终答案是否正确 → 最主要的 reward
- recall 行为是否有效（召回内容是否与最终答案相关）→ 辅助 reward
- memory 更新质量 → 辅助 reward

这三层 reward 在 advantage 层面聚合，同时优化记忆更新和 recall 两种行为。

## 六、你的毕设如何借鉴这个流程

你不需要重新训练模型，借鉴思路如下：

| ReMemR1 原始机制 | 你的毕设对应实现 |
|---|---|
| 长文档分块读取 + memory update | 考研政策文档分块 RAG 检索 |
| `<recall>` 触发历史检索 | 多轮对话中检测是否需要回溯历史提问 |
| TF-IDF 检索 history_memory | 直接复用 `tf_idf_retriever.py` |
| memory 覆盖更新 | 对话状态记忆（用户目标/偏好）动态更新 |
| TEMPLATE / TEMPLATE_FINAL_BOXED | 改写为考研问答场景的 Prompt 模板 |

核心区别：ReMemR1 是在**单次长文档阅读**中 recall 历史片段；你的毕设是在**多轮对话**中 recall 历史轮次的关键信息。这个场景扩展就是你的创新点。
